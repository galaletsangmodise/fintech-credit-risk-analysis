FinTech Credit Risk Analysis 
Project Overview 
This project analyzes loan data to identify credit risk patterns in a fintech lending environment. The goal is to understand which borrower characteristics increase the likelihood of loan default and how lenders can make better strategic decisions using data.  
Business Problem 
FinTech lenders face significant risk from loan defaults. By analysing borrower financial characteristics and loan outcomes, companies can identify high-risk segments and adjust lending strategies accordingly. 
Tools Used
- SQL Server
- SQL
- GitHub
Dataset
The dataset contains loan records including:
- loan amount
- credit grade
- annual income
- loan purpose
- employment length
- debt-to-income ratio 
- loan status (Fully Paid or Charged Off) 
Key Questions
1. What is the overall default rate of the loan portfolio?
2.  Which credit grades have the highest risk?
3. Do lower income borrowers default more often?
4. Which loan purposes show the highest default rates?

Key Findings
The dataset shows a 50% default rate.
Grade C borrowers represent the largest share of risky loans.
Lower income borrowers appear more likely to default.
Personal loans show the highest number of defaults. 

Strategic Insights 
FinTech lenders could reduce risk by: 
- tightening approval criteria for Grade C borrowers
-  introducing minimum income thresholds
-  limiting high-risk loan categories
- improving credit risk assessment models
